# vectorSum
Каждый поток видеокарты получает задачу сложить ровно один пару элементов векторов A и B. Количество потоков зависит от элементов векторов. Каждый элемент C[idx] вычисляется на основе: элемента A[idx] и элемента B[idx]. При этом результат для C[0] никак не зависит от результата для C[1] или любого другого элемента. Это значит, что все элементы можно вычислять одновременно и независимо. Вся задача разбивается на блоки, каждый блок — на потоки. Параметр BLOCK_SIZE = 256 означает, что один блок содержит 256 потоков, организованных линейно. Количество блоков вычисляется как   
(n + BLOCK_SIZE - 1) / BLOCK_SIZE, чтобы покрыть весь вектор. Каждый поток определяет свой глобальный индекс: idx = blockIdx.x * 256 + threadIdx.x. Затем поток выполняет сложение A[idx] + B[idx], записывая результат в C[idx], но только если idx не превышает размер вектора, что защищает от выхода за границы.


## эксперимент
первый столбик - размерность; второй столбик - время выполнения на cpu, ms; третий столбик - время выполнения на gpu, ms
